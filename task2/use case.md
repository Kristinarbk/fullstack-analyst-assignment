User Story

**Действие:** Как продавец маркетплейса я хочу иметь возможность
добавлять свои товары на витрине маркетплейса.

**Цель:** Чтобы покупатели могли найти и приобрести мой товар.

**Acceptance criteria:**

1.  В личном кабинете продавца есть кнопка «Добавить товар».

2.  Форма создания товара содержит все обязательные поля: Название,
    Описание, Категория, Количество на складе, Цена и блок загрузки
    изображений.

3.  Ограничения для изображений: формат jpeg/png, количество от 1 до 10
    штук, размер одного файла не более 5 МБ.

4.  Реализована валидация полей: система подсвечивает ошибки и блокирует
    публикацию, если не заполнены обязательные поля или есть введены
    некорректные данные (цена\<=0, количество на складе \<0, не
    соблюдены требования к изображениям).

5.  Реализована кнопка «Сохранить как черновик» (товар не уходит на
    модерацию, название товара должно быть заполнено обязательно).

6.  Перед публикацией товар проходит модерацию (статус «На модерации»).

7.  При прохождении модерации статус товара меняется на «Опубликован».
    Продавец получает уведомление в личном кабинете.

8.  После прохождения модерации товар становится доступен для поиска и
    отображается на витрине маркетплейса.

9.  При отклонении модератором статус товара меняется на «Отклонен».
    Продавец получает уведомление в личном кабинете с причинами
    отклонения.

10. Для товара в статусе «Черновик» или «Отклонен» продавец может
    отредактировать данные карточки товара (для статуса «Отклонен»
    причины отклонения видны в карточке) и нажать «Опубликовать». После
    этого товар вновь переходит в статус «На модерации».

Use Case

UC-001: Создание карточки товара и отправка на публикацию

Акторы: Продавец, Система маркетплейса

Предусловия: Продавец авторизован в Личном кабинете (ЛК).

> Триггер: Продавец нажимает кнопку «Добавить товар»

Основной поток:

+----+-------------------------------+---------------------------------+
| Ш  | Продавец                      | Система                         |
| аг |                               |                                 |
+====+===============================+=================================+
| 1  | В ЛК переходит в раздел       | Открывает форму создания товара |
|    | управления товарами и         | и присваивает статус «Новый».   |
|    | нажимает кнопку «Добавить     |                                 |
|    | товар».                       |                                 |
+----+-------------------------------+---------------------------------+
| 2  | Заполняет все обязательные    | Отображает введенные данные и   |
|    | поля карточки и загружает от  | загруженные файлы в интерфейсе. |
|    | 1 до 10 изображений.          |                                 |
+----+-------------------------------+---------------------------------+
| 3  | Нажимает кнопку               | 1.  Проводит валидацию данных   |
|    | «Опубликовать».               |     (заполненность полей,       |
|    |                               |     цена\>0, остаток\>=0,       |
|    |                               |     размер и формат файлов.     |
|    |                               |                                 |
|    |                               | 2.  Сохраняет товар в БД,       |
|    |                               |     меняет его статус на «На    |
|    |                               |     модерации».                 |
|    |                               |                                 |
|    |                               | 3.  Отображает сообщение «Товар |
|    |                               |     отправлен на модерацию» и   |
|    |                               |     возвращает пользователя в   |
|    |                               |     раздел управления товарами. |
+----+-------------------------------+---------------------------------+

Постусловие: Товар сохранен в БД и ему присвоен статус «На модерации».

Альтернативные потоки:

**А-1. Сохранение товара как «Черновик» (из 3 шага)**

3.1 Продавец вместо кнопки «Опубликовать» нажимает кнопку «Сохранить как
черновик».

3.2 Система проверяет заполнено ли поле «Название».

3.2.1 Если не заполнено: Система подсвечивает поле «Название» красным,
выводит ошибку «Заполните название для сохранения черновика» и блокирует
действие. Возврат на шаг 2 Основного потока.

3.3 Система сохраняет товар в БД со статусом «Черновик».

3.4 Система выводит сообщение «Черновик успешно сохранен» и
перенаправляет продавца в раздел управления товарами.

**А-2. Редактирование существующего товара со статусом «Черновик» или
«Отклонен» (из шага 1)**

1.  Продавец в разделе управления товарами выбирает товар со статусом
    «Черновик» или «Отклонен» и нажимает кнопку «Редактировать».

2.  Система открывает форму редактирования с ранее заполненными данными.

3.  Если товар в статусе «Отклонен», то система дополнительно выводит
    блок с текстом причины отклонения в верхней части формы.

4.  Продавец вносит изменения и сценарий переходит к шагу 3 Основного
    потока.

**А-3. Ошибка валидации данных, введены некорректные данные (из шага
3)**

3.1 Система обнаруживает некорректные данные (цена\<=0, остаток\<0 или
несоответствие требованиям изображений).

3.2 Система блокирует отправку формы.

3.3 Система подсвечивает ошибочные поля красным цветом и выводит
соответствующие подсказки под каждым полем (например, «Цена должна быть
больше 0»).

3.4 Продавец исправляет ошибки и сценарий возвращается к шагу 3
основного потока.

Use Case

UC-002: Модерация карточки товара

Акторы: Модератор, Система маркетплейса

Предусловия: Карточка товара находится в статусе «На модерации».

> Триггер: Модератор открывает раздел очереди модерации.

Основной поток:

  ------------------------------------------------------------------------
  Шаг     Модератор                      Система
  ------- ------------------------------ ---------------------------------
  1       Открывает очередь модерации    Отображает список карточек
                                         товаров «На модерации»

  2       Открывает карточку товара и    Отображает форму просмотра
          проверяет на соответствие      карточки товара.
          правилам маркетплейса.         

  3       Нажимает «Одобрить»            Меняет статус карточки товара на
                                         «Опубликован», публикует товар,
                                         отправляет уведомление продавцу:
                                         «Ваш товар успешно опубликован» и
                                         перенаправляет модератора в
                                         раздел очереди модерации.
  ------------------------------------------------------------------------

Постусловие: Товар опубликован и отображается на витрине и в поиске.

Альтернативный поток:

А-1. Модератор отклоняет товар (из шага 3)

3.1 Модератор нажимает «Отклонить».

3.2 Система требует ввести причину (текстовое поле или выбрать причину
из справочника).

3.3 Модератор вводит причину отклонения и подтверждает действие.

3.4 Система меняет статус карточки товара на «Отклонен»

3.5 Система отправляет уведомление продавцу: «Товар не прошел проверку
модерации. Причина: \[Текст причины отклонения\]» и сценарий
возвращается к шагу 1 основного потока.
